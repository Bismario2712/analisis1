Definiciones

Análisis de sistema: Es el proceso de estudiar y descomponer un sistema en sus componentes fundamentales para comprender su funcionamiento, identificar problemas y proponer mejoras. Este análisis se enfoca en entender cómo interactúan los diferentes elementos del sistema y cómo contribuyen al objetivo general del mismo.

Concepto de análisis: El análisis es el proceso de examinar detalladamente los componentes o elementos de un objeto, fenómeno o situación para entenderlo mejor. Implica la descomposición del todo en sus partes más pequeñas para estudiar sus relaciones y funcionamiento interno.

Sistema: Un sistema es un conjunto de elementos interrelacionados y que interactúan entre sí para lograr un objetivo común. Estos elementos pueden ser personas, máquinas, procesos o conceptos, y el sistema puede ser tan simple como una bicicleta o tan complejo como una organización empresarial.

Proyecto: Un proyecto es un esfuerzo temporal llevado a cabo para crear un producto, servicio o resultado único. Tiene un inicio y un final definidos, y se realiza con el propósito de cumplir objetivos específicos mediante la aplicación de recursos y la gestión de actividades planificadas.

Planificación: La planificación es el proceso de definir objetivos y determinar las acciones necesarias para alcanzarlos. Incluye la identificación de recursos, la asignación de tareas, la previsión de posibles obstáculos y la creación de un cronograma para garantizar que las metas se cumplan de manera eficiente y efectiva.

Ejemplos de aplicación

Identificación de Requisitos: Recopilar y documentar las necesidades del software mediante entrevistas, encuestas y análisis de documentos.

Modelado de Sistema: Crear diagramas y modelos como diagramas de flujo de datos y diagramas de casos de uso para visualizar el sistema.

Análisis de Factibilidad: Evaluar la viabilidad del proyecto considerando tecnología, economía, tiempo y recursos humanos.

Definición de Arquitectura del Sistema: Determinar la estructura del sistema, incluyendo plataformas tecnológicas, lenguajes de programación y bases de datos.

Especificación de Requisitos: Crear un documento detallado que describa las funcionalidades, interfaces de usuario y requisitos del sistema.

Relaciones y conexiones

Definición y Alcance: El análisis de sistemas es una etapa crucial dentro de un proyecto de sistema. Durante el análisis de sistemas, se define el alcance del proyecto, se identifican los requisitos del sistema y se establecen los objetivos. Esto proporciona una base sólida para el desarrollo del proyecto de sistema.

Identificación de Requisitos: El análisis de sistemas ayuda a identificar y documentar los requisitos funcionales y no funcionales del sistema. Estos requisitos son esenciales para diseñar y construir el sistema dentro del proyecto.

Modelado y Diseño: A partir del análisis, se desarrollan modelos y diagramas que representan el funcionamiento del sistema. Estos modelos guían el diseño y la implementación del sistema durante el proyecto.

Ventajas y desventajas

Ventajas:

  • Proporciona una dirección clara.
  • Previene problemas potenciales.
  • Optimiza el uso de recursos.
  • Facilita el monitoreo y control del progreso.

Desventajas:

  • Puede ser un proceso largo y detallado.
  • Requiere información precisa y actualizada.
  • La mala planificación puede llevar al fracaso del proyecto.

Estudio de caso

Escenario Ficticio: La Biblioteca Universitaria "BiblioTech" decide modernizar sus operaciones con un nuevo sistema de gestión bibliotecaria (SGB). Este sistema debe permitir a los estudiantes y al personal universitario buscar libros, gestionar préstamos y devoluciones, reservar salas de estudio y acceder a recursos digitales. Además, el sistema debe integrar funcionalidades para el personal bibliotecario, como la gestión de inventarios, adquisiciones y catalogación.

Aplicación del Análisis de Sistema

Identificación de Requisitos:

  • Usuarios:
    • Estudiantes y personal universitario: Búsqueda de libros, gestión de préstamos y reservas de salas.
    • Personal bibliotecario: Gestión de inventarios, adquisiciones, catalogación y administración del sistema.
  • Funcionalidades Clave:
    • Catálogo en línea para búsqueda de libros.
    • Sistema de gestión de préstamos y devoluciones.
    • Reservas de salas de estudio.
    • Acceso a recursos digitales (e-books, artículos, etc.).
    • Herramientas de administración para el personal bibliotecario.
  • Requisitos No Funcionales:
    • Seguridad: Autenticación y autorización de usuarios.
    • Usabilidad: Interfaz amigable y fácil de usar.
    • Rendimiento: Capacidad de manejar múltiples solicitudes simultáneas.

Modelado de Sistema:

  • Diagrama de Flujo de Datos (DFD): Muestra el flujo de información entre usuarios (estudiantes, personal universitario y bibliotecarios) y el sistema.
  • Diagrama de Casos de Uso: Ilustra las interacciones entre los diferentes actores y las funcionalidades del sistema (búsqueda de libros, gestión de préstamos, etc.).
  • Diagrama de Entidad-Relación (ERD): Define las entidades (libros, usuarios, préstamos, reservas) y sus relaciones.

Análisis de Factibilidad:

  • Tecnológica: Evaluar las tecnologías disponibles (bases de datos, frameworks de desarrollo web).
  • Económica: Análisis de costo-beneficio para asegurar la viabilidad financiera del proyecto.
  • Operacional: Determinar si el personal bibliotecario está capacitado para usar el nuevo sistema.

Definición de Arquitectura del Sistema:

  • Decidir entre una arquitectura basada en microservicios o una aplicación monolítica.
  • Seleccionar plataformas tecnológicas: Frontend (React, Angular), Backend (Node.js, Django), Base de datos (MySQL, PostgreSQL).

Especificación de Requisitos: Crear un documento detallado que describa todas las funcionalidades, interfaces y requisitos técnicos del sistema.

Aplicación de la Planificación

Definición de Objetivos:

  • Desarrollar un SGB que cumpla con todos los requisitos funcionales y no funcionales identificados.
  • Completar el proyecto en un plazo de 12 meses con un presupuesto de $200,000.

Creación de un Plan de Trabajo:

  • Fase de Análisis y Diseño (2 meses): Finalización del análisis de sistema y modelado, especificación de requisitos.
  • Fase de Desarrollo (6 meses): Codificación del sistema, desarrollo de la base de datos, implementación de funcionalidades.
  • Fase de Pruebas (2 meses): Pruebas unitarias, de integración y de aceptación.
  • Fase de Implementación (2 meses): Despliegue del sistema, capacitación del personal, migración de datos.

Asignación de Recursos:

  • Equipo de desarrollo: 5 desarrolladores, 2 testers, 1 diseñador UX/UI, 1 administrador de base de datos.
  • Infraestructura: Servidores, herramientas de desarrollo y prueba.
